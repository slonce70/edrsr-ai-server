# 🔄 Система повторного выполнения заданий

## 📋 Обзор проблемы

**До исправления:** Задания со статусом `error` никогда не выполнялись повторно и оставались "зависшими" навсегда.

**После исправления:** Умная система автоматического и ручного retry для временных ошибок.

## 🎯 Типы ошибок

### ✅ **Временные ошибки (автоматический retry)**
- `Memory limit exceeded` - переполнение памяти
- `Worker terminated due to reaching memory limit` - аварийное завершение
- `зависла` - зависание задачи
- `timeout` - таймауты
- `превысил` - превышение лимитов времени
- `network`/`ENET`/`ECONN` - сетевые ошибки
- `503`/`502` - временные ошибки сервера
- `fetch failed` - ошибки загрузки

### ❌ **Критические ошибки (НЕ повторяются)**
- Неверные URL
- Отсутствие прав доступа
- Некорректные данные
- Системные ошибки конфигурации

## 🔧 Автоматические механизмы

### 1. **Периодическое восстановление**
- **Частота:** Каждые 5 минут
- **Условия:** 
  - Статус `error`
  - Попыток < 3
  - Возраст задания < 24 часа
  - Тип ошибки - временный

### 2. **Восстановление при старте**
- Автоматически запускается при старте сервера
- Восстанавливает stuck и failed задания

## 🎛️ API endpoints (Admin)

### **GET** `/api/admin/jobs/errors`
Получить список заданий с ошибками

**Parameters:**
- `limit` (optional, default: 20) - количество заданий

**Response:**
```json
{
  "success": true,
  "jobs": [
    {
      "id": "job-id",
      "status": "error",
      "error_message": "Memory limit exceeded: 450MB > 400MB",
      "attempt": 1,
      "updated_at": "2024-01-15T10:30:00Z",
      "created_at": "2024-01-15T10:00:00Z"
    }
  ]
}
```

### **POST** `/api/admin/jobs/:jobId/retry`
Перезапустить конкретное задание

**Response:**
```json
{
  "success": true,
  "message": "Задание поставлено на повторное выполнение"
}
```

### **POST** `/api/admin/jobs/retry-failed`
Автоматическое восстановление всех заданий с временными ошибками

**Response:**
```json
{
  "success": true,
  "message": "Перезапущено 3 заданий с временными ошибками",
  "retried_count": 3
}
```

## 📊 Обновленная статистика дашборда

Новые метрики в админ дашборде:
- **Total Jobs** - общее количество заданий
- **Completed Jobs** - успешно завершенные
- **Failed Jobs** - с ошибками
- **Retryable Jobs** - задания с ошибками, которые можно повторить (< 3 попыток)

## 🔍 Логирование

### **Автоматическое восстановление:**
```
🔄 [RECOVERY] Восстановлено 2 зависших заданий в БД
🔄 [RETRY] Повторяется 3 заданий с временными ошибками
```

### **Мануальный retry:**
```
🔧 Manual retry initiated for job abc-123
```

### **Admin действия:**
- `VIEW_ERROR_JOBS` - просмотр заданий с ошибками
- `RETRY_JOB` - ручной retry задания
- `RETRY_FAILED_JOBS` - массовый retry

## 🚀 Использование

### **Для администраторов:**

1. **Просмотр проблемных заданий:**
   ```bash
   GET /api/admin/jobs/errors?limit=50
   ```

2. **Ручной retry конкретного задания:**
   ```bash
   POST /api/admin/jobs/abc-123/retry
   ```

3. **Массовое восстановление:**
   ```bash
   POST /api/admin/jobs/retry-failed
   ```

### **Автоматические процессы:**
- При старте сервера автоматически восстанавливаются задания
- Каждые 5 минут проверяются failed задания на retry
- Логи показывают активность системы восстановления

## 🛡️ Защиты

1. **Лимит попыток:** максимум 3 попытки на задание
2. **Время жизни:** retry только заданий младше 24 часов  
3. **Фильтрация:** retry только временных ошибок
4. **Логирование:** все действия записываются в audit log

## ✅ Ожидаемый результат

После деплоя ваши 4 задания с ошибками:
1. **Автоматически проверятся** при старте сервера
2. **Перезапустятся** если ошибки временные (memory, timeout, network)
3. **Останутся в error** если ошибки критические

**Проверьте логи** после деплоя - должны увидеть сообщения о восстановлении!
