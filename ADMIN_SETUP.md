# Настройка административной панели EDRSR-AI

## Быстрый старт

### 1. Настройка переменных окружения

В файле `server/.env` добавьте или проверьте наличие следующих переменных:

```env
# Supabase настройки (обязательно)
SUPABASE_URL=https://your-project.supabase.co
SUPABASE_ANON_KEY=your-anon-key
SUPABASE_SERVICE_ROLE_KEY=your-service-role-key

# База данных (уже настроено)
DATABASE_URL=postgresql://...

# Сервер
PORT=4000
```

### 2. Создание первого администратора

После настройки переменных окружения, создайте первого администратора:

```bash
cd server
node scripts/create-admin.js your-admin-email@example.com
```

Скрипт автоматически:
- Инициализирует таблицы базы данных
- Создаст пользователя (если не существует) 
- Назначит права администратора
- Покажет временный пароль (если пользователь новый)

### 3. Запуск сервера

```bash
cd server
npm run dev
```

### 4. Доступ к админской панели

Откройте в браузере: `http://localhost:4000/admin`

Введите email и пароль администратора для входа.

## Функции административной панели

### 📊 Дашборд
- Общая статистика системы
- Количество пользователей и заданий
- Использование памяти
- Время работы сервера

### 👥 Управление пользователями
- Просмотр всех зарегистрированных пользователей
- Назначение/отзыв прав администратора
- Поиск пользователей по email

### 📋 Управление заданиями
- Просмотр всех заданий всех пользователей
- Фильтрация по статусу
- Удаление заданий
- Поиск заданий

### ⚙️ Системные операции
- Очистка старых данных
- Статистика производительности
- Мониторинг системы

### 📝 Аудит действий
- Журнал всех административных действий
- IP-адреса и временные метки
- Детальная информация о операциях

## Безопасность

### Контроль доступа
- Только пользователи с ролью 'admin' имеют доступ
- Все действия логируются
- JWT токены для авторизации

### Права администратора
- Администратор не может отозвать права у самого себя
- Все изменения отслеживаются в audit log
- Возможность назначать других администраторов

## Добавление новых администраторов

### Через админскую панель:
1. Пользователь должен сначала зарегистрироваться в расширении
2. В админской панели перейти в "Пользователи"
3. Найти пользователя и нажать "Сделать админом"

### Через скрипт:
```bash
node scripts/create-admin.js new-admin@example.com
```

```env
SUPABASE_SERVICE_ROLE_KEY=eyJ...

```

## Устранение неполадок

### Проблема: "Supabase not configured"
**Решение:** Убедитесь что все переменные Supabase настроены в `.env`

### Проблема: "У вас нет прав администратора"
**Решение:** Запустите скрипт создания администратора:
```bash
node scripts/create-admin.js your-email@example.com
```

### Проблема: "Ошибка подключения к базе данных"
**Решение:** Проверьте правильность `DATABASE_URL` в `.env`

### Проблема: Админская панель не загружается
**Решение:** 
1. Убедитесь что сервер запущен на правильном порту
2. Проверьте что статические файлы находятся в `server/public/admin/`
3. Откройте консоль браузера для просмотра ошибок

## Архитектура

```
server/
├── public/admin/          # Статические файлы админки
│   ├── index.html         # Главная страница
│   ├── styles.css         # Стили
│   └── script.js          # JavaScript функциональность
├── routes/admin.js        # API endpoints для админки
├── middleware/adminAuth.js # Middleware проверки прав

```

## API Endpoints

- `GET /api/admin/dashboard` - Статистика дашборда
- `GET /api/admin/users` - Список пользователей
- `POST /api/admin/users/:id/make-admin` - Назначить администратора
- `DELETE /api/admin/users/:id/admin` - Отозвать права
- `GET /api/admin/jobs` - Список всех заданий
- `DELETE /api/admin/jobs/:id` - Удалить задание
- `POST /api/admin/system/cleanup` - Системная очистка
- `GET /api/admin/audit-log` - Журнал аудита

Все endpoints требуют авторизацию с JWT токеном и права администратора.
